.hyphens-as-word-separators

  • include a single space at the end of the selector, before the opening brace of a ruleset.
  • place the closing brace of a ruleset in the same column as the first character of the ruleset.
  • group selectors that share a pseudo-class on a single line, but otherwise, place discrete selectors on their own line in multi-selector rulesets.
  • include only one declaration per line in a declaration block.
  • use one level of indentation for each declaration.
  • include single space after the colon of a declaration.
  • include a space after each comma in comma-separated property or function values.
  • use lowercase hex values for colors, shorthand if possible, unless using rgba().
  • quote attribute values in selectors, e.g., input[type="checkbox"]. Exceptions to this rule include urls in background and font-face declarations.
  • 0 values should not have units unless necessary, such as with transition-duration.
  • include leading zeroes on decimal values.
  • avoid qualifying class names with type selectors.
